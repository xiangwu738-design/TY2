# 《同渊》内核 MVP（Godot 4.7）

这是一个**只验证内核**的最小可玩原型。目标是先把规则骨架里最核心、最独特的那几条
机制跑起来、能上手感觉到手感，再在此基础上删改。联机、附魔、拼刀、血量上限、
roguelike 外壳等**都没有做**（见文末“待删改点”）。

## 怎么跑

1. 用 Godot 4.7（4.x 均可）打开本目录（`project.godot`）。
2. 主场景已设为 `main.tscn`，直接 F5 运行。
3. 无任何外部依赖，纯 GDScript。

## 操作

- 鼠标**悬停**任意角色的【手牌 / 整备 / 空过】：
  - 时间轴上**黄色**=该动作将推进的格子，**红⚠**=将触发的敌人节点；
  - 底部文字显示预计后果（途经节点、队友将受伤、敌 HP 变化、是否结束）。
  - 这是“完全信息预读”（规则 §3 预打出预览）的最小实现。
- **单击**即执行该动作。
- 顶部【新局】重开（固定种子 12345，可改 `main.gd` 的 `SEED`）。

## 单局说明（MVP 范围）

- 单人操控固定四角色（甲乙丙丁），共用一条时间轴、一个指针。
  出牌权自由分配（规则 §9：出牌权是全队共担的稀缺资源）。
- 敌人没有牌堆，它的攻击是**铺在时间轴上的节点**，指针经过即触发（规则 §5）。
- 核心手感：**占位越高，效果越迟落地，沿途越可能先挨打**（规则 §1 关键语义）。
  你能在出牌前看到“我打这张重击（占3）会路过 2 个敌人节点、先挨 14 伤才落地”。
- 位置系统（规则 §3）：出攻击牌 → 移到 1 位（输出即暴露），身前的人后移。
  敌人斩击打 1 位、突刺打 2 位、打击打全体。靠“换谁上前线”来躲伤害。

## 工程结构

```
scripts/core/        纯逻辑内核（RefCounted，不依赖节点，可单测、可克隆、可预演）
  enums.gd           枚举
  card.gd            牌定义（不可变 + effect 回调）
  card_defs.gd       起手牌组 / 敌人编排
  character.gd       角色状态 + 牌堆操作
  enemy.gd           敌人 = 关卡编排
  game_state.gd      ★ 核心：时间轴推进 + 逐格结算 + 预演 + 克隆
scripts/ui/main.gd   表现层：只渲染 + 转发动作，不持有规则逻辑
main.tscn            根场景
```

**内核与表现严格分层**：`GameState` 是确定性的，`preview_action()` 在克隆状态上
跑一遍就能给出“松手前”的完整后果。将来联机（主机权威 + 动作同步，§9）只需把
`play_card/use_prep/skip` 的调用通过 RPC 转发、事件流广播即可，内核不动。

## 内核 ↔ 规则骨架 对应

| 规则条 | 实现 |
|---|---|
| §1 时间轴单层格子 + 逐格结算 | `GameState._advance()`：进格→触发节点→（终点）落牌效果 |
| §1 占位=时间成本 | `CardDef.occupancy`，攻击牌落点在末尾、沿途先挨打 |
| §2 出牌不抽牌 / 整备 / 空过 | `play_card` / `use_prep`(弃1抽2) / `skip` |
| §2 手牌起5上限8 / 弃牌堆洗回 | `Character.HAND_START/LIMIT`、`draw()` |
| §3 位置系统 / 输出即暴露 / 敌三型 | `_move_to_front()`、`_trigger_node()` |
| §3 预打出预览 / 数值预览 | `preview_action()` + UI 悬停高亮 |
| §5 敌人=关卡编排（谱子） | `Enemy.pattern` 循环铺格 |
| §6 防御（走位/治疗） | `move_back`、`heal_char`；护盾未做 |

## 待删改点（你验证后我们逐项决定）

- **附魔（§4）**：未做。当前牌是共享不可变定义；做附魔需引入“每张牌的独立实例 +
  附魔载体”，这是内核下一个最大的扩展点。
- **护盾 / 守护关系（§5/§6）**：未做。防御目前只有走位 + 治疗。
- **血量上限长期损伤（§7）**：未做。当前每局回满、无跨场。
- **拼刀流 / 破绽节点（§6）**：未做（进阶，最后再上）。
- **roguelike 外壳（§8）**：未做（地图/商店/移牌/遗物）。
- **联机（§9）**：未做。内核已为动作同步预留。
- **重洗随机**：弃牌堆洗回目前用“反转”代替随机洗（保持确定性便于调试），
  属输入端随机，正式版可换带种子的洗牌。
- **平衡数值**：所有伤害/血量/节点密度都是占位，规则明确“只能原型调”。

## 建议的验证顺序（对应规则 §10 验证路径）

1. 先确认 §1 的“逐格结算 / 高占位先挨打”手感对不对（悬停重击看预览）。
2. 确认 §3 位置系统的“换人上前线躲斩击”是否成立、是否好玩。
3. 再决定附魔、护盾、血量上限往哪加。
