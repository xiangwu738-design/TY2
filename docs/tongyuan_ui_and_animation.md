# 《同渊》UI 与动画设计文档（整合版）

> 本文整合最新落定的布局、立绘、演出、队列与动画方案,取代早先那版"敌人占顶部整条"的旧布局。
> 一句话基调:**经典回合制 RPG 的视觉外壳 + 流畅手感优先的演出**(已明确取舍:牺牲一点厚重演出感,换节奏流畅,详见 §九)。

---

## 一、风格定调
- **视觉**:经典回合制 RPG —— 深墨基底 + 暖金描边的窗口面板 + 厚重衬线 / 宋体字 + 宝石四色 + 明确光标。结合《同渊》气质(深渊、东方厚重),偏八方旅人 / 女神异闻录的暗色精致,非明快卡通 JRPG。
- **演出取向**:**流畅手感 > 华丽厚重**。逐格立绘演出是系统支持的"上限",不是每张牌的"下限";绝大多数牌快速过,厚重演出留给极少数高光时刻。

---

## 二、屏幕布局（最新）

上半部左右分(我方立绘 / 敌人),中段时间轴,底部手牌:

```
┌──────────────────────────────┬───────────────────────────┐
│  我方四人立绘（左→右 = 4/3/2/1 位）│   敌人立绘                  │
│   [4位][3位][2位][1位]         │   敌人名 · HP ▓▓▓▓ 60/60    │
│   各色 · 待机/施法/重伤 立绘     │   [易伤2] [虚弱1]           │
│   1 位若被瞄准 → 警示高亮        │   下个意图：斩击 → 1位 · 12  │
└──────────────────────────────┴───────────────────────────┘

  时间轴  格0 ═◆指针═ 格1  格2 ◇破绽  格3 ◆敌·斩12  格4 …
          └─（红·拖牌预览）          队列： A的牌 → B的牌 → …

┌────────────────────────────────────────────┬───────────┐
│ 手牌（尖塔式扇形持握）[突刺][招架][突刺][破绽]  │  ◆ 整备   │
│   ↑上拖 = 打出     ↓下拖 = 弃为固定牌           │ 抽12 弃3  │
└────────────────────────────────────────────┴───────────┘
```

- **我方立绘在左上**,**排列顺序即站位**(左→右 4/3/2/1,1 位最靠中、最前线、最靠敌人侧)。
- **敌人在右上**(立绘 + HP + 标记 + 意图)。
- **时间轴横贯中段**:指针、敌人节点(明牌)、破绽节点、拖牌预览、队列。
- **手牌在底**,**尖塔式扇形持握**;右侧常驻"整备"按钮 + 牌堆计数。

## 三、立绘系统

### 状态立绘（按角色，优先做）
- 每个角色一套:**待机 / 施法 / 受击 / 重伤** 等。
- 数量可控(4 角色 × 几个状态)、复用率极高(整局都在用),是**美术地基,优先级最高**。
- 重伤立绘对应血量 / 重伤机制,让"角色状态"在画面上看得见。

### 技能演出立绘（按牌，节制做）
- **占位 = 演出帧数**:占位 2 的技能可配 2 张不同立绘(逐格各一张),占位 4 = 4 帧。演出长度天然锚定占位,与"占位 = 时间成本"同构——贵牌演得长。
- **一图多用**:多帧不必是多张全新原画,可用同一立绘的姿态变化 + 特效层(蓄力光 → 挥击光)制造"多帧"感。
- **美术分级**:绝大多数牌 = 一帧立绘 + 简单特效、快速过;**逐格多帧的厚重演出 = 少数招牌大招的特权**(见 §四 与 §九)。

### 站位变化的演出
- 出攻击牌 → 该角色移到 1 位;被挤的角色**播一个后撤帧再滑动移位**,让换位是看得见的动作,而非数据跳变。

## 四、演出与队列系统

- **即时出牌 → 牌进队列 → 指针逐张走过、逐张结算**(类杀戮尖塔2 联机的队列,但本作单卡更重,故需下面的节制)。
- **关键解法:只有"指针正经过的当下那张"播演出,排队中的牌只显示轻量预览(牌面 / 缩略)**。→ 同一时刻只有一份演出在播,**等待感恒定、不随队列长度累积**。这是"既要厚重演出、又要队列流畅"这对矛盾的调和点。
- **快速演出为基础**:以"占位 = 帧数"为节奏基准、快速过。**加速是基础设定,不是可选开关**。
- **厚重演出 = 稀有高光特例**:仅限极少数时刻(如 Boss 击杀最后一击、终极技、拼刀成功那一下),触发标准要严,稀缺才有分量。
- **队列硬上限**:用规则把队列焊短(如同时未结算牌 ≤ 2,或每人未结算牌 ≤ 1),防失控;不靠"自然不会长"的乐观。
- **敌人行动明牌铺在时间轴上**:玩家排牌时,敌人节点何时触发、干什么,明明白白在同一条时间轴上 → **排队时仍能完整预读敌人的中途插入**,保住完全信息(把早先"敌人意图明牌"升级为"敌人行动也排在时间轴上")。

## 五、跨区系统

- **预打出预览**(完全信息 + 协作核心):任意玩家拖牌(未松手)→ 全体可见牌面 + 该玩家色的占位预览(推进到哪、跨过哪些格)+ 攻击牌"将移到 1 位"预览。松手才结算。
- **数值预览**(尖塔式):伤害显示实时含修正(附魔 / 易伤 / 虚弱),玩家看到的永远是实际生效数字(如 6 → 9),旁注"已含易伤"。
- **四色系统**:四名玩家固定四色,全局一致——立绘描边、HP 条、占位预览、手牌卡描边。一眼认人。
  - 玩家1 朱红 `#D4453A` / 玩家2 湛蓝 `#3D7BD4` / 玩家3 翠绿 `#3DA862` / 玩家4 琥珀 `#E0A82E`。
- **敌人标记**:敌人身上的状态图标(易伤 / 虚弱…),是**附魔的只读显示器**——数据真相在附魔,标记只反映、不另存(见规则骨架 §4)。

| 窗口面板 | `#1E2530`(半透明) | 各区 window box |
| 描边 / 描金 | `#C8A858` | 暖金边框 |
| 主文字 | `#ECE4D2` | 羊皮纸暖白 |

| 敌人 / 危险 | `#A32D2D` | 敌人 HP、敌人节点 |

| 破绽 / 暴击 | `#EF9F27` | 亮金破绽节点 |
| 护盾 | `#5DCAA5` | 青白 |

## 七、动画清单（接《打击感提示词》）

- **指针推进**:逐格平滑滑动(每格约 0.12s,ease out);跨过有节点的格时轻顿一下(踏节点感)。
- **立绘演出**:指针进入牌的每一格 → 切到该格对应立绘(占位 = 帧数);仅当下结算的牌演出,排队牌轻量预览。
- **站位移动**:被挤角色后撤帧 → 滑动到新位;出牌角色滑到 1 位。
- **受击**:hit stop 短定格 + 闪白 + 击退 + 屏震 + 伤害数字上飘(护甲 / 扣血不同色,大伤害大字号)。
- **被瞄准脉动**:被锁定的位置 / 立绘,警示色缓慢呼吸,预示威胁临近。
- **卡牌拖拽**:拖起放大 + 倾斜;上拖飞向目标后消失;下拖向下滑出、变灰、碎裂淡出。
- **加速 / 压缩演出**:基础即为快速;高光时刻才放完整厚重演出。
- 具体数值(定格时长、震幅、缓动)留原型手调。

## 八、用 UI 对抗信息过载（本作 UI 的第一要务）

经典 RPG 的窗口美学正是解药——靠位置和窗口分担认知,而非心算:
1. **该记的全可视化**:占位、节点、附魔(敌人标记)、护盾、被瞄准位,全部画出来。完全信息靠"看得见",不靠"算得出"。
2. **数值预览替玩家算**:修正后的真实数字直接显示。
3. **分区固定**:立绘左上、敌人右上、时间轴中、手牌下——位置恒定,形成肌肉记忆。
4. **四色 + 分工降载**:四色让"谁是谁"零思考;可按角色职责弱化非己信息,让认知在四人间分摊。
5. **预览先行、松手才结算**:看清后果再提交,试错成本归零。

## 九、设计取舍记录（诚实备注）

- **已明确牺牲"厚重逐格演出",换"流畅手感"**:这是有意识的取舍,不是疏漏。后果是——**这游戏的"好看 / 记忆点"不能再指望演出去扛**,得交给别处:那条独特的时间轴本身、四色协作的画面、UI 整体质感。设计与美术资源应据此分配。
- **厚重演出降为稀有特例**:务必严控触发,别泛滥(泛滥则美术量回到爆炸、且大招不再特殊)。
- **这一整套(布局 / 立绘 / 队列 / 演出 / 动画)本质都是手感**:快慢、厚薄、队列顺不顺,逻辑推不出答案,**只有占位图原型跑一局能告诉你**。文档把取舍和结构定清楚了,接下来该上手验,而非继续纸上细化。

## 十、给 Claude Code 的实现提示

```
做四人卡牌战斗 UI + 演出,经典回合制 RPG 风格(深墨基底、暖金描边窗口、宋体厚重字、宝石四色、明确光标),不要现代扁平风。

布局:
- 左上:我方四人立绘,排列顺序即站位(左→右 4/3/2/1),各用玩家专属色;有待机/施法/受击/重伤状态立绘;1位被瞄准则高亮。
- 右上:敌人立绘 + HP + 标记(附魔只读显示器) + 意图(类型+瞄准位+伤害，含修正)。
- 中:共享时间轴,指针/敌人节点(明牌)/破绽节点;拖牌时叠占位预览;队列显示。
- 下:手牌尖塔式扇形持握,右侧常驻"整备",牌堆计数。

演出与队列:
- 即时出牌→进队列→指针逐张走过结算。只有"当下指针正经过的那张"播立绘演出,排队牌只显示轻量预览(保证同一时刻只一份演出、不累积)。
- 占位=演出帧数(占位2配2帧立绘),快速过为基础;厚重多帧演出仅限少数高光(Boss击杀/终极技/拼刀成功)。
- 队列设硬上限防失控。敌人行动明牌铺在时间轴上,让玩家排牌时能预读敌人中途插入。
- 站位变化:被挤角色播后撤帧再滑动移位。

跨区:预打出预览(全体可见)、数值预览(实时含修正)、四色全局一致。
配色字体见文档色板;动画见打击感清单。
信息呈现:一切可视化、预览替算、分区固定、四色降载。
```
