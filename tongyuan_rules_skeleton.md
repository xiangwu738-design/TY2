# 《同渊》规则骨架（当前版）

> 这份是当前最新落定的规则整合。被推翻的旧版(疲劳、自我格挡、按格数的状态印记、暴击率掷骰)不再出现。
> 标「核心·已验」= 原型验证过;「核心·待验」= 已定方向但没玩过;「进阶」= 流派性内容,不是人人必碰的基础。

---

## 0. 定位
- **四人协作(联机)+ 单人**,固定四角色,单多人共用同一套规则。
- **完全可决策**(由"完全信息"松绑而来):玩家做决策的那一刻,信息足够算清后果。**随机允许存在于输入端**(抽到什么牌、关卡出什么题),**但玩家出牌后的结果不靠事后掷骰子**——尤其敌人,用"可预读的分支"而非"事后随机"来制造变化。
- roguelike 局内构筑(尖塔式外壳)。基调:**低挫败、温暖小队协作**。

---

## 1. 时间轴 —— 单层格子 + 逐格结算 「核心·已验」

- **时间轴 = 一串格子**。指针停在格子上、从左向右**逐格推进**。废除"间隙"层,**一切都挂在格子上**。
- **占位 = 推进的格数 = 时间成本**(取代费用 / 能量)。占位锁定铁律:占位 0 只能设计师写死,不可由减费效果动态产生。
- **格子上能放两类东西**:
  - **事件点**(经过即触发一次):敌人行动节点、破绽节点(见 §6 进阶)。
  - **关系持续态**:护盾(绑"守护关系",见 §5)。
  - (注:状态类 buff 不再铺在时间轴上,已改为"牌的附魔",见 §4。)
- **逐格结算顺序**(消除一切跨多格歧义)——指针每**新进入**一格(出发格不重复结算):
  1. 触发该格的事件点;
  2. 检查该格的持续态;
  3. 若该格是本牌占位的终点 → **结算本牌效果**。
- **关键语义**:牌效果在占位**末尾**才落下,而沿途敌人节点先触发 → **打牌时,敌人可能抢在你的效果生效前先出手**。占位越高、效果越迟落地、途中越可能先挨打。

---

## 2. 出牌与手牌运转 「核心·已验」

- **打牌**:推进占位 → 末尾结算效果 → 打出的牌进**弃牌堆**。**出牌不抽牌**(打一张少一张,手牌是消耗资源)。抽牌堆抽空 → 弃牌堆整叠洗回。
- **固定整备技能**:常驻、不占手牌、可重复使用;负责**运转手牌**(如弃 1 抽 2),本身花时间(推进占位)。是手牌补给的主要途径。
- **空过**:只推进时间,永远可用(防死锁)。
- 手牌:起手 5,上限 8。
- **核心呼吸**:进攻(打牌耗手牌)↔ 整备(运转补手牌),两者都花时间。

---

## 3. 位置系统 「核心·待验」

- 四位,从左到右 **4 / 3 / 2 / 1**,1 位最前线。四名玩家各有**专属颜色**。
- **出攻击牌 → 该玩家移到 1 位**,其身前的人依次后移(**输出即暴露**)。
- **敌人攻击三型**:斩击=打 1 位、突刺=打 2 位、打击=打全体。被瞄准位置 + 其上玩家**预先高亮**。
- **预打出预览**(完全信息 + 协作核心):任意玩家拖牌(未松手)时,所有人可见——该牌是什么 + 占位预览(用其颜色,显示推进到哪、跨过哪些格)+ 攻击牌"将移到 1 位"的预览。松手才结算。
- **数值预览**:所有伤害预览实时反映当前修正(附魔、敌我状态),玩家看到的永远是实际生效数字。

---

## 4. Buff = 附魔（作用于“牌”，不作用于抽象数值层）「核心·待验」

- **所有 buff 抽象为"附魔"**,载体是**牌**这个实体,与本作"以牌为核心货币"统一。不再有"按张数 / 按格数 / 分三类"的结算难题——因为它不经过抽象结算层,它就是牌。
- **玩家侧**:附魔加在玩家的牌上(可作用于手牌 / 弃牌堆 / 抽牌堆)。例:"为弃牌堆所有牌增伤" → 玩家要**在牌堆的混沌中找秩序、主动控牌**去兑现它。这是"被驯服的随机",是手艺,不是骰子。
- **敌人侧**:敌人的 buff 加在**敌人的牌**上(敌人没有抽弃牌堆,其牌是关卡编排,见 §5)。敌我用同一套附魔语言,对称。
- **标记 = 附魔的显示器(分清里子与面子)**:**附魔是机制载体**(管 buff 怎么生效、作用在谁的牌上);**敌人身上的标记是显示层**(把"与该敌人相关的附魔状态"汇总成一个看得见的图标,补上"纯附魔藏在牌里、敌人状态不可见"的盲区)。关键约束:**标记不持有独立数据,只反映附魔这一个真相源**——其层数 / 时效由所联动的附魔决定(例:附魔是"接下来 3 次对它的攻击增伤",标记就显示 3,每打一次减一,归零消失)。机制层统一(只有附魔)、显示层直观(状态画在敌人身上),两层分工、不重复、不互相存数据。
- **随机原则**:附魔带来的随机在**输入端**(你抽到、堆到什么),玩家用打牌顺序去**驾驭**它——决策的结果仍是确定可算的。

---

## 5. 敌人 = 关卡（编排 + 可预读分支）「核心·待验」

- 敌人**不是玩家镜像**,没有牌堆循环。它的"牌"是**关卡设计师预先编排、铺在时间轴上的序列**(像谱子),玩家走到触发。
- **节点布局是关卡语言**:节点的疏密 / 前后,编码敌人的性格与攻击节奏(快攻 = 密且靠前;成长 = 疏且靠后)。**平衡主旋钮在节点布局,而非牌的数值**(类《陷阵之志》)。
- **分支,而非随机**:优秀关卡**对玩家行动做分支**——敌人会因你的站位 / 血量 / 行为切换行为,但**切换条件是明牌、可预读的**(你出牌前就知道"我若如此、它便如彼")。这给出"活对手、不背板"的全部好处,又不破坏可决策性。
- **少量输入端随机**(敌人组合、序列变体、初始站位)用于重玩性;但**不在玩家决策后用不可见骰子改变结果**(尤其警惕"伤害值小幅浮动"这种最廉价、却最伤害精确计算的随机)。
- **护盾**:绑"守护关系"——为某个位铺盾,**只要被守护者仍在那位,盾就在;他一离位(阵型乱了),盾消失**。

## 6. 防御体系 + 拼刀流

**基础防御三条腿** 「核心·待验」(已去掉尖塔式自我格挡):
- **走位规避**:预读敌人打哪个位,别站那(第一选择,不挨打最好)。
- **护盾**:他人布防,绑位置 / 守护关系(见 §5)。
- **治疗**:善后,接住漏掉的伤害(配合 §7 血量)。

**拼刀(进阶 / 流派,非基础)**:
- 曾考虑当基础机制,因"玩家伤害与敌人攻击几乎不可能持续匹配"而**降为一个流派**——匹配从"系统要替所有人保证"变成"这个流派主动构筑去够"。
- 规则:**攻击牌(占位 ≥ 2)经过敌人节点 → 比伤害**。赢 → 完全无效化该次攻击 + 生成**破绽节点**;输 → 全额挨打、不抵扣(全有全无)。占位 0/1 的弱牌不能拼。Boss 大招等标为**不可拼**。
- **破绽节点 = 本作的"暴击"**(条件触发的确定爆发,零随机):拼赢后生成,任何队友的攻击牌经过 → 吃伤害加成 →"一人拼赢开窗、众人扑上"的协作连携。

---

## 7. 血量体系 「核心·待验」

- **个人血**(非共享)。
- **每场开局回满当前血** → 不带残血进场,守住低挫败地板、不滚雪球。
- **战斗结算时,按受伤程度下调"血量上限"** → 提供跨场的、有分量的长期损伤资源;**上限只能靠休息恢复**(给"休整 / 不参战"一个具体数值意义)。
- **被击倒 → 加 debuff**:仅**本场内**的临时削弱(必须"弱而不废",不致坐牢拖累),**本场结束清除**;长期代价由"上限下调"独自承担,两者不叠。
- 上限一路被压低 → 通向**重伤 → 天堂线**(叙事死亡阶梯,见设计文档叙事节)。
- **命门**:"上限掉多快 vs 休息回多易" 这对速率,是本机制成败的全部所在 —— 调到"每次掉上限都肉疼、又不会一路单向崩盘"的甜区。**只能原型调。**

---

## 8. 构筑（roguelike 外壳）「核心·待验」

- 尖塔式结构:地图、战后加牌、商店、移牌、遗物。单多人共用。
- **成型卡组偏小(约 15 张,12–18 区间)**:因"出牌不抽 + 整备快循环 + 完全信息要可算 + 四人出牌权稀缺",机制天生奖励"小而精"。起手单薄(~12)、成型长到 15–18。
- **获取牌偏受控**(近 Chrono Ark 而非尖塔慷慨),配"质量保底 + 移牌顺畅"软化挫败。
- 构筑主轴是**占位 / 时间**(而非能量曲线)。遗物挂占位 / 时间轴机制(守占位下限)。

---

## 9. 四人协作要点

- **共享时间轴**:四人共用一条轴、一个指针,出牌权是**全队共担的稀缺资源**(段长决定总量)。
- 故四人牌走**"少而重" + 协作增益 / 连携**(我的牌让你更狠),把"出得少"变成"必须配合"。
- **认知分工**:四人不必各盯全部信息,可让不同角色主理不同维度(输出 / 布防 / 读意图),让协作顺便成为复杂度的分摊器。
- **联机**:主机权威 + 同步动作(非状态);localhost 多窗口 → 局域网 → 公网中继。玩法验证先用本地共享屏幕,别让联机工程卡住玩法。

---

## 10. 诚实备注（复杂度账本 & 验证路径）

- **复杂度提醒**:这套系统已相当庞大,且最近每解决一个设计问题,几乎都是用"再加一个机制"来解(附魔、拼刀流、血量上限……)。单个都自洽,但**总复杂度的增长本身,是比任何单点都更该警惕的趋势**。建议每加一项时,主动问一句"它配得上我要的简单吗"。
- **验证路径(一层只验一个变量)**:单人核心循环【已验】→ 接入真实战斗到联机 → 加位置系统(四人协作"好不好玩"从此开始被回答)→ 真人测"垄断 / 疯狂"等开放问题 → 血量上限速率手调 → 拼刀流等进阶最后再上。
- **尚未经原型的关键手感点**:① 血量"掉上限 / 休息恢复"速率;② 位置系统下的四人协作;③ 破绽节点的位置与时效;④ 附魔制在实战中的认知负担。这些纸上都给不了答案,必须做出来玩。
